如何搭建一个独立博客，简明Github Pages与Hexo教程

弄了好久，终于还是成功了，踩过很多很多坑，正因为如此，才打算整理（包括别人写的总结一下）一个教程。把自己踩过的坑与一点点经验分享下。

大概步骤：
1.买一个域名
2.安装好git和Node.js（不多说，自己下载）
3.配置SSH keys
4.github上建立仓库
5.用Hexo克隆主题
6.安装theme
7.将独立域名与GitHub Pages的空间绑定
8.DNS设置解析

大概步骤就这么多。只要按顺序做好一般是不会出问题的。

一：注册域名
本人是在新网注册的（可根据自己喜欢的网站去注册）就说下大概步骤，先注册登录，在设计自己喜欢的域名，查询看看是否已被注册，如果找到合适的域名加入购物车去购买，新用户一年38元，之后每年60吧，千万别选其他服务。
二：
1.$ brew install node 安装Node.js
2.$ brew cask install git 安装Git
3.查看安装的版本，检验是否安装正确
$ node -v

$ npm -v

$ git –version

三：配置SSH keys

四.github上建立仓库
1.在github首页右上角+号—>New Repository 添加一个空项目
2.Repository name:名字
Description 介绍
注：Repository名字是特定的，比如我Github账号是timesaid，那么我Github Pages Repository名字就是timesaid.github.io。
3.点击Create resitory创建成功

五.用Hexo克隆主题
*安装Hexo，执行以下指令

1.$ npm install -g hexo

创建一个目录，用于放置你的Hexo博客，然后通过终端进入你创建的目录，初始化站点

2.$ hexo init hexo

初始化站点后，系统会提示执行npm install以完成所有的配置

3.$ npm install

上面操作完成后，即完成Hexo的安装和初始化，
*hexo文件夹目录结构
source：博客资源文件夹
source/_drafts：草稿文件夹
source/_posts：文章文件夹
themes：存放主题的文件夹
themes/landscape：默认的主题
_config.yml：全局配置文件
4.部署到Github Pages
终端cd到blog文件夹下，vim打开_config.yml，命令如下：

(1)$ vim _config.yml

打开后往下滑到最后，修改成下边的样子：
deploy:
type: git
repository: https://github.com/timesaid/timesaid.github.io.git()
branch: master
你需要将repository后timesaid换成你自己的用户名
注意：在配置所有的_config.yml文件时（包括theme中的），在所有的冒号:后边都要加一个空格，否则执行hexo命令会报错，切记 切记
(2)在blog文件夹目录下执行生成静态页面命令：

$ hexo generate

此时若出现如下报错：
ERROR Local hexo not found in ~/blog
ERROR Try runing: ‘npm install hexo –save’
则执行命令：
npm install hexo –save
若无报错，自行忽略此步骤
(3)再执行配置命令：

$ hexo deploy

(4)注意：若执行命令hexo deploy仍然报错：无法连接git或找不到git，则执行如下命令来安装hexo-deployer-git：

$ npm install hexo-deployer-git –save

之后在执行
$ hexo generate
$ hexo deploy
或直接使用组合键
$ hexo d -g

(5)hexo deploy命令执行成功后，浏览器中打开网址http://timesaid.github.io（将timesaid换成你的用户名）能看到效果

5.发布文章
(1)终端cd到blog文件夹下，执行如下命令新建文章：

$ hexo new “timesaid”

(2)文章编辑完成后，终端cd到blog文件夹下，执行如下命令来发布：
$ hexo d -g
至此，Mac上搭建基于Github的Hexo博客就完成了。下面的内容是介绍安装theme，添加评论功能和绑定个人域名

六.安装theme
你可以到”Hexo官网主题页”去搜寻自己喜欢的theme。这里以hexo-theme-time为例
(1)终端cd到 blog 目录下执行如下命令：
$ git clone https://github.com/MOxFIVE/hexo-theme-yelee themes-time

(2)将blog目录下_config.yml里theme的名称landscape修改为time(要不然看不到效果)

(3)终端cd到blog目录下执行如下命令(每次部署文章的步骤)：
$ hexo d -g

注意：至于更改theme内容，比如名称，描述，头像等去修改blog/_config.yml文件和blog/themes/time/_config.yml文件中对应的属性名称即可， 不要忘记冒号:后加空格。

七.将独立域名与GitHub Pages的空间绑定
我的做法是：github —>timesaid.github.io —>ADD a README(下面会提示) ———>将README改为CNAME—->文件里写入自己的域名（我的为例：www.timesaid.com）切记，我因为没加www.导致网页一直打不开报错
注意：可能每次提交都会让CNAME消失，解决办法，下载这个项目—>找到CNAME—>将文件放到blog下的public文件中。这样每次提交都不会出问题了。

八.DNS设置解析
1.（以我在新网注册为例）登录—>我的产品 —>解析
注意：先要实名认证，（mac用户的safari和谷歌点击解析是不起作用的，windows的360浏览器可以，这块有问题可咨询新网客服），如果新网有同事加你qq什么的，让你备份空间需要钱的一概不理，如果觉得自己做麻烦除外。
2.实名认证后要将实名的模板换上。（一般24小时左右）
3.获取自己的ip：
$ ping timesaid.github.io(timesaid自己github的名字)会打印出自己的ip。
3.解析界面—>主机名A最多允许20条—>前面写www—>自己的徐明—>填写自己的域名ip：（例151.101.16.132）解析一般会在4-—48小时之间。

4.以上完成，看看github—>点击项目（timesaid.github.io例）—>Settings—>查看GitHub Pages(如果是对勾且显示 Your site is published at http://www.timesaid.com/)
恭喜你完成了！！！！

注意：如果警告可能是CNAME没弄好。当然没解析成功哪里会显示timesaid.github.io（timesaid自己的名字）也可以进入自己的博客。。
